# Feature: CLI Archive Listing (`--list`)

## 1. Overview
Implement the `-t` / `--list` command to display the contents of an existing `quiver` archive directly in the terminal. This command provides a quick, read-only overview of the packed files without extracting them. It acts as the CLI frontend for the internal metadata API (`namelist` / `infolist`) and must correctly handle embedded archives (ignoring PREAMBLE and EPILOGUE).

## 2. Requirements

### 2.1. CLI & Input
* **Command:** `quiver -t -f <existing_archive.xml>`
* **Aliases:** `--list` for `-t`.
* **Behavior:** Reads the specified archive and prints the contained file paths to standard output (`stdout`).
* **Flags:**
    * **Default (Silent/Standard):** If only `-t` is provided, print a flat, line-by-line list of the normalized POSIX paths (e.g., `src/main.py`). This allows the output to be easily piped into other Unix tools like `grep`.
    * **Verbose (`-v` / `--verbose`):** If the verbose flag is provided, use the `rich` library to print a formatted table showing extended metadata (e.g., File Path, File Size).
    * **Debug (`--debug`):** Activate internal `structlog` logging as usual.

### 2.2. Core Logic: Read-Only Parsing
* **Embedded Archive Support:** The listing logic must scan for the first `<archive>` block boundaries, completely ignoring any `PREAMBLE` or `EPILOGUE` text surrounding the XML.
* **API Integration:** Instantiate the `AsyncQuiverFile` in read mode. 
    * For the default output, use `aqf.namelist()`.
    * For the verbose output, use `aqf.infolist()` to access the `size` property of the `QuiverInfo` objects.
* **Tree Independence:** The list must be generated by iterating over the actual `<file path="...">` attributes, **not** by printing the `<directory_tree>` string. This ensures the output reflects the absolute truth of the packed data.

### 2.3. Expected Output Format
**Standard Output (Default):**
```text
src/main.py
src/utils/helper.py
tests/test_main.py
```

**Verbose Output (with `-v`):**
*(Rendered via `rich` Table)*
```text
┏━━━━━━━━━━━━━━━━━━━━━┳━━━━━━━━━━━┓
┃ File Path           ┃ Size      ┃
┡━━━━━━━━━━━━━━━━━━━━━╇━━━━━━━━━━━┩
│ src/main.py         │ 1.2 KB    │
│ src/utils/helper.py │ 450 B     │
│ tests/test_main.py  │ 2.1 KB    │
└─────────────────────┴───────────┘
```

## 3. Acceptance Criteria (Testing)
All tests must run asynchronously (`pytest-asyncio`), use `pyfakefs`, and enforce strict typing.

* [ ] **Test Standard Listing:** Running `-t` outputs exactly the correct file paths separated by newlines, with no additional formatting or UI clutter.
* [ ] **Test Verbose Listing:** Running `-t -v` triggers the `rich` table generation and includes correct file sizes.
* [ ] **Test Embedded Listing:** The command successfully lists files even if the archive is heavily wrapped in `PREAMBLE` and `EPILOGUE` text.
* [ ] **Test Empty/Invalid Archive:** Running the command on a file without an `<archive>` tag exits with a clear, handled error and a non-zero exit code.
* [ ] **Type Checking:** All CLI routing and API consumption passes `mypy --strict` without errors.
